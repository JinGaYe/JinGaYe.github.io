    # 1. 初始化 vuepress 项目

    • 全局安装 VuePress
    yarn global add vuepress
或者
npm i vuepress -g
    • 创建项目目录
    mkdir myblog
cd myblog
yarn init -y 或者 npm init -y
//带上-y使用默认跳过
    • 新建 docs 文件夹
      • docs 文件夹作为项目文档根目录，主要放置 markdown 类型的文章和.vuepress 文件夹。
    mkdir docs
    • 设置 package.json
      • 主要是设置运行指令和打包指令及后续的部署指令
    {
  "scripts": {
    "dev": "vuepress dev docs --temp .temp",
    "build": "vuepress build docs",
    "deploy": " ./deploy.sh"
  }
}

    • 其中，--temp .temp是使用缓存运行，除非增删文件才会报错，否则改了什么内容都可以不用重新运行就能看到，类似webpack的热更新

    • 创建.vuepress 目录

      • 在 docs 目录中，创建.vuepress 目录，.vuepress 目录主要用于存放 VuePress 相关的文件。
    mkdir .vuepress
    • 创建 config.js
      • 进入到.vuepress 目录中，然后创建 config.js，config.js 是 Vuepress 必要的配置文件，它导出一个 javascript 对象
    touch config.jjs
    • 创建 public 文件夹
      • 进入到.vuepress 目录中，然后创建 public 文件夹，此文件夹主要放静态资源文件，例如 favicons 和 PWA 的图标。
    mkdir public
    • 此时，项目结构大致如下：
    vuepress-theme-sakura
|————————docs(后续markdown与自定义的组件放在这下面)
|       |——————README.md
|       |_________ .vuepress
|             |——————public
|             |_________config.js
|——————package.json
|_________deploy.sh

    # 2. 初始化 vuepress 主题

    • 由于对 vuepress 还不是很熟悉，刚开始写自己主题的话，可以先把默认的主题拉下来，再进行修改
    cd docs/.vuepress
vuepress eject
//执行完毕后就可以看到多出来theme文件夹，里面即是初始的默认主题，拉取后就可以在上方进行修改了

    # 3. 修改默认主题

    • 有了上一步之后，这一步，你可以在 theme 里的 components 写你自己的组件，但注意这些组件无法直接引用到 md 中，可以改掉布局和样式等。
      • 视自己情况而定，这里简单写了一个调模式的组件：
    <template>
  <div class="mode-switch">
    <el-switch
      v-model="mode"
      active-color="#FFCC66"
      inactive-color="#6495ED"
      active-value=" "
      inactive-value="dark-mode"
      @change="changeMode"
      active-icon-class="iconfont icon-sun"
      inactive-icon-class="iconfont icon-moon"
    >
    </el-switch>
  </div>
</template>

<script>
  export default {
    name: 'DarkMode',
    data() {
      return {
        mode: ' ',
      };
    },
    mounted() {
      this.IsDayOrNight();
      setInterval(() => {
        this.IsDayOrNight();
      }, 1000 * 60 * 120);
    },
    methods: {
      changeMode(mode) {
        const html = document.getElementsByTagName('html')[0];
        html.setAttribute('theme', mode);
      },

      IsDayOrNight() {
        const hour = new Date().getHours();
        if (hour > 5 && hour < 19) {
          this.mode = ' ';
        } else {
          this.mode = 'dark-mode';
        }
        this.changeMode(this.mode);
      },
    },
  };
</script>

    # 4. 自定义组件

    • 如果要是自己的组件直接使用在 md 中，需直接在.vuepress文件夹下创建components编写自己的组件
    • 示例如下(限于篇幅，只显示时间线这个内容较少的组件)：
    <template>
  <div class="blog-timeline">
    <el-timeline reverse>
      <el-timeline-item
        v-for="(page, index) in pages"
        :key="index"
        type="success"
      >
        <RouterLink :to="page.path" replace>
          <el-card>
            <p class="title">{{ page.frontmatter.title }}</p>
            <p v-show="page.frontmatter.keywords" class="keywords">
              {{ page.frontmatter.keywords }}
            </p>
            <p class="description" v-show="page.frontmatter.description">
              {{ page.frontmatter.description }}
            </p>
            <p class="date">{{ page.frontmatter.date }}</p>
          </el-card>
        </RouterLink>
      </el-timeline-item>
    </el-timeline>
  </div>
</template>

<script>
  export default {
    name: 'BlogTimeline',
    computed: {
      pages() {
        const {pages} = this.$site;
        const unsortPage = pages.filter(page => {
          page.frontmatter.date = this.$parseTime(page.frontmatter.date);
          return page.frontmatter.post;
        });
        unsortPage.sort((a, b) =>
          a.frontmatter.date > b.frontmatter.date ? 1 : -1
        );
        return unsortPage;
      },
    },
  };
</script>

    # 5. 增强 vuepress 功能

    • 增强 vuepress 的功能，包括使用第三方 ui 或者是功能性增强等
      • 在docs/.vuepress下创建enhanceApp.js，我的配置如下：
    /**
 * 扩展 VuePress 应用
 */
import ElementUI from 'element-ui';
import 'element-ui/lib/theme-chalk/index.css';
import * as Tool from './theme/util/tool';
export default ({Vue, options, router, siteData}) => {
  // ...做一些其他的应用级别的优化
  Vue.use(ElementUI); //引入elementUI
  // 引入Vue的原型方法
  Object.keys(Tool).map(key => {
    Vue.prototype[`$${key}`] = Tool[key];
  });
  Vue.mixin({
    mounted() {
      setTimeout(() => {
        try {
          document &&
            (() => {
              Tool.copy(); //加入复制内容的功能
            })();
        } catch (e) {
          console.log(e.message);
        }
      }, 500);
    },
  });
};

    # 7. 引入插件

    1. vuepress-plugin-helper-live2d
      • 右下角看板娘
    2. vuepress-plugin-sitemap
      • 站点地图
    3. vuepress-plugin-baidu-autopush
      • 自动推送百度站点
    4. vuepress-plugin-rss
      • 生成 rss 订阅
    5. vuepress-plugin-element-ui
      • 直接 md 中使用 element-ui 组件
    6. vue-social-share
      • 该组件在开发环境使用没问题，如果使用一般的方法，在打包就会报错，原因是在 node 环境下找不到window与document对象，此处使用 vuepress 提供的动态组件方法避免该问题，具体操作如下：
    <template>
  <div class="right-options">
    <component v-if="share" :is="share" :config="config" />
  </div>
</template>

<script>
  export default {
    name: 'RightOptions',
    data() {
      return {
        share: null,
      };
    },
    computed: {
      config() {
        return {
          url: this.url,
          disabled: ['google', 'facebook', 'twitter', 'diandian', 'douban'],
        };
      },
    },
    props: {
      url: [String],
    },
    mounted() {
      import('vue-social-share').then(m => {
        //  关键是延迟引入的做法
        this.share = m.default;
      });
    },
  };
</script>

    # 8. 部署到 github page

    echo "正在构建中"

echo "构建完毕"
echo "进入打包目录"
cd docs/.vuepress/dist
echo '域名' > CNAME
echo '初始化git'
git init

echo '添加文件'
git add -A
git commit -m 'deploy'

echo '部署推送到github page'
git push -f git@github.com:username/username.github.io master
    • 保存为deploy.sh，之后，运行./deploy.sh即可部署到 github page 上
